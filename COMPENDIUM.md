# Continuity-of-Order: Honest Compendium of the Current Work

## Abstract

This compendium records the current state of a Continuity-of-Order (COT) theorem-candidate program. The working hypothesis is that ordered path history can expose transition information that pointwise, aggregate, or static summaries can miss. The project has been tested or scaffolded across turbulence, transition-front, fracture, plasma, protein-folding, Planck/action-phase, and DESI/vacuum-energy diagnostic branches.

The current result is not a theorem. It is a structured evidence program with strict promotion gates, observed-data rules, failure logs, data-wall requests, and reproducibility packets. The public purpose of this package is to preserve the work honestly enough that another reader can see what was attempted, what passed locally, what failed, and what exact evidence is still required.

## Working Thesis

COT should be treated as a disciplined validation framework until it earns more. It asks:

> Does ordered history contain a reproducible transition signal that survives independent data, strong baselines, wrong-form mutations, uncertainty gates, and strict provenance?

The answer is currently:

> Not proven. Promising enough to keep testing, but blocked before theorem-grade promotion.

## What COT Is

COT, in this package, means **Continuity of Order**. It is not a claim about private reasoning or hidden inference. It is a proposed way to organize observed data into path/history-aware diagnostic tests.

Core objects:

- **Path:** an ordered sequence of observed states or source-followed samples.
- **Growth:** a measured change along a path under a frozen rule.
- **Transition:** a predeclared change in behavior, regime, support, or stability.
- **Failure boundary:** the precise condition where the COT claim stops being supported.
- **Promotion:** movement up an evidence ladder only after locked gates pass.

## The Evidence Ladder

The active ladder distinguishes recovery from theorem evidence:

- **L5:** multi-benchmark recovery or known-result reproduction.
- **L6:** heldout recovery with clean separation from construction/tuning data.
- **L6+:** heldout recovery with diagnostic utility.
- **L7:** distinct prediction or distinct mechanism survival.
- **L8+:** broader theorem-grade support across independent settings.

The current public package claims no L6/L7 promotion.

## What Has Been Learned

The work has not yet produced a theorem, but it has produced useful negative structure:

- Same-source recovery can look strong while still failing independent-source promotion.
- Target payloads are not enough without source provenance.
- Point-path evidence cannot be promoted to region-support evidence.
- Sparse front anchors cannot be promoted to geometric front motion.
- A local signal can be absorbed by strong baselines.
- Wrong-form and mutation survival are serious blockers, not details.
- Claim language must be locked as carefully as code or data.

- It records the failures without burying them.
